6. Format of the File to Upload

The Wall Manager allows users to upload their applications, which can use different platforms, and make them run on the Wall. Since each platform has its own way of launching applications, there is are some specifications you need to be aware of.

Specifications

Zip Bundle

The application should be packed as one zip (application/zip mime-type) with all the files inside. This allows a much more simple upload and the zip compression makes it fast to do so.

Initializer

The multi-touch computer is running a Windows XP as its operative system. A simple way of allowing developers to customize the way their application is launched is through a Batch File: a text file with a sequence of commands that is executed by the command prompt and a .bat extension.

This solution has the flexibility of allowing users to write their own initialization scripts in other language supported by the wall (Python, for instance) by just writing a one-line batch file.

Contents

You should add your application files to the zip. This includes the binaries (C, C++, Java) or the source code (Python) and resources used by them (Images, etc...)

Structure

YourApplication.zip should contain the following files:

  • boot.bat (the mandatory initializer)
  • Other files your application requires.

Restrictions

Your application should be self-contained. Except for system libraries or provided platforms (such as JDK and MT4J, OpenFrameworks, Python and PyMT) all the dependencies of your application should be inside your zip file. After the zip file is unpacked, your application should only need to write in that directory. This makes the uninstall process more efficient.

Security

Each application is connected to a student's account, and the information of what application is being launched is logged and WILL be used.
