4. Restrictions

There are some details that the developer needs to know, specific to the wall, in order to make it run like it's supposed to. This section's objective is minimize the amount of unpredicted behaviors after the application is deployed, so that the end user experiences it the way both the wallmanager team and the users planned it.

The Exit Gesture

One of the things you must not forget, is the "reserved" exit method the SenseWall has implemented.

There are two ways to end an application running on the SenseWall:

  • 1. Each application is expected to have its own quit functionality in the usual places: an option in the main menu or a button near the screen corners.
  • 2. The universal way to end an application is to perform a special gesture (the Exit Method) anywhere in the SenseWall screen. This gesture will quit the application in every situation, even on freezes or crashes, overriding it completely. The quit gesture is:


The Exit Gesture


In this sense, it is advised that the programmer doesn't make use of this specific gesture (the Exit Method), or another too similar, in their own applications.

Gracefull Exits

The "Exit Method" has already been implemented and made available to all running applications. All that's left to ask of developers is to make their applications tolerant to failures with a "gracefull exit" so as to avoid nasty application crashes.

Please read the following link to learn more about "Gracefull Exits":
http://en.wikipedia.org/wiki/Graceful_exit

FullScreen

An other thing to remind yourself, is that the wall applications will probably run in fullscreen mode.

That is why developers must bear that in mind and make their applications ready for running in fullscreen mode on the SenseWall.

ScreenSaver Applications

Know that, by submitting an application of the ScreenSaver category type, it can be executed in a slightly different way than the usual applications.

Ideally, by touching an screen saver application it should close. However, the developer may want to add interactive elements, like dragging or resizing, but the larger majority of the screen saver applications should close after a single touch.

Who defines what's the current screensaver are the SenseWall administrators. If you wish, you can contact him to ask to have your screen saver running at the SenseWall for some time. If you're not sure how to do it, please read section Contact Administrator of the User Documentation.

Folders

Finally, please make use of the correct folders in your application. Namely, please respect the following directives:

  • Use the folder that the application will be running for temporary files.
  • Use the %USERDATA% folder for permanent data, even after application updates.
